## Reportar uma vulnerabilidade

Se você descobrir uma vulnerabilidade de segurança neste projeto, agradecemos sua colaboração.

**Por favor, não abra uma issue pública** para reportar vulnerabilidades de segurança.

### Como reportar

1. Envie um e-mail para o mantenedor do projeto descrevendo:
   - O tipo de vulnerabilidade
   - Passos para reproduzir
   - Impacto potencial
   - Sugestões de correção (se houver)

2. Aguarde uma resposta em até **48 horas**.

3. O mantenedor confirmará o recebimento e trabalhará em uma correção.

4. Após a correção, a vulnerabilidade será divulgada de forma responsável (sem detalhes de exploração antes do patch).

### O que esperar

- Resposta inicial em até 48 horas
- Atualização sobre o progresso da correção
- Crédito ao descobridor (se desejar) na nota de divulgação

### Escopo

Este projeto é um CLI local que usa o GitHub CLI para consultar a API do GitHub. Não armazena credenciais diretamente. Vulnerabilidades relacionadas a:

- Exposição de tokens/credenciais
- Injeção de código
- Execução remota não autorizada

são de alta prioridade.
